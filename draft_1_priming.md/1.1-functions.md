# functions
- in haskell a function is a mapping that takes in one or more arg and ruturns a single results
    the fuction will have naming
    the args will have naming
    and the retunr will also have naming

    eg: a funtion for performing x + x would be
        equation:
        double x = x + x

- in case when an argument is another functions
    the given function will be simplified down until it becomes and axiom ( number )

- continuing the example
    double function that takes in x and returns x + x
    double x = x + x

    // application
    double 3
    3 + 3
    6

    // application 2
    double ( double 2 )
    double ( 2 + 2 )
    double ( 4 )
    4 + 4
    8


    // application 3 - keep in mind that one operation is one double
    double ( double 2 )
    double 2 + double 2
    2 + 2 + double 2
    4 + double 2
    4 + 2 + 2
    4 + 4
    8


- Issues:
    the order of functions simplification do on affect the results
    but it does affect the number of steps it takes

    so you have to take in account expression evaluation order
